# 从 Webpack 移植到 Turbopack

我们打算让 Turbopack 称为 Webpack 的接班人，在未来为 Turbopack 提供各种工具来支持你的 Webpack 应用。

目前，暂不支持从 Webpack 移植到 Turbopack。未来，我们计划为所有的 Webpack 用户提供丝滑的移植体验。

## Turbopack 会兼容 Webpack 的 API 么？

Webpack 拥有很多 API，这让它能够进行灵活配置，也是它为什么这么出名的主要原因。

我们打算让 Turbopack 变得十分灵活可拓展，但我们没打算让它能够 1 比 1 兼容 Webpack。这样我们就能选择哪些 Webpack API 可以针对速度和效率上进行优化。

## Turbopack 可以使用 Webpack 的插件么？

Webpack 插件对于整个 Webpack 生态环境是至关重要的。通过提供底层工具，你可以使用插件自定义工具链来最大化提升生产力。

由于我们不打算为 Webpack 提供 1 比 1 的兼容，所以几乎 Webpack 插件都不能直接在 Turbopack 运行。

但是，我们正在将一些知名的插件移植到 Turbopack。
